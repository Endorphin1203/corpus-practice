# 语料练习系统 — 设计文档

> 日期：2026-07-17 | 状态：已完成

## 1. 项目概述

### 1.1 背景

用户积累了两份英语写作语料 Excel（描写续写 + 议论文），共 11 个分类、约 500+ 条中英对照语料。考试场景下，需要先有中文逻辑再转化为英文，因此需要一个系统来训练这种"中→英"的转换能力。

### 1.2 目标

构建一个 Web 练习系统，基于语料库自动生成多种题型，通过 AI 精细纠错反馈，帮助用户将被动积累转化为主动运用能力。

### 1.3 核心功能

- 三种题型：翻译题、语境选择题、场景写作题
- 两种练习模式：日常积累（随机抽取）、考前突击（优先薄弱项）
- AI 精细纠错：语法检查 + 用词分析 + 改进版本
- 完整学习追踪：正确率趋势、分类统计、薄弱项标注
- 语料管理：Excel 批量导入 + 界面内增删改
- 多 AI 后端支持：国产大模型（OpenAI 兼容接口）+ 本地模型

## 5. 模块详细设计

### 5.1 语料管理模块

**Excel 导入：**
- 使用 Apache POI 解析 .xlsx
- 根据 Sheet 名称自动映射 category/subcategory
- 导入策略：追加（不覆盖已有数据），按 `chinese` 去重
- 前端显示导入进度和结果摘要（新增 X 条，跳过 Y 条重复）

**界面管理：**
- 列表页：支持按 category/subcategory 筛选、关键词搜索
- 单条增删改：对话框表单，chinese/english/notes 三个字段
- 批量选择删除

### 5.2 题目生成引擎

**三种题型：**

| 题型 | 生成方式 | AI 参与 | 说明 |
|------|---------|---------|------|
| 翻译题 | 从语料库随机抽取中文 | ❌ 不需要 | 直接取 `chinese` 字段作为题目 |
| 语境选择题 | AI 生成 | ✅ 必须 | 1 个正确语料 + 3 个 AI 生成的干扰项 |
| 写作题 | AI 生成 | ✅ 必须 | AI 生成写作场景，指定运用的语料数量/类别 |

**生成流程：**
1. 用户配置：选择分类 → 各题型数量 → 练习模式
2. 翻译题：直接从选中分类中随机抽取
3. 选择/写作题：拼接 prompt → 调用 AI → 解析 JSON 返回 → 存入 exercise_cache
4. AI 生成的题目缓存 24 小时，期间再次请求相同参数直接复用

**考前模式差异：**
- 语料抽取权重根据历史正确率调整：错误越多的语料被抽中概率越高
- 不是全部按错误率，保留 30% 随机以维持覆盖面

### 5.3 练习引擎

**答题流程：**
```
展示题目 → 用户作答 → 提交 → AI 精细纠错 → 展示反馈 → 下一题
```

**翻译题界面：**
- 上方：中文原文
- 中间：英文输入框（textarea，自适应高度）
- 下方：提交按钮 + 计时器

**选择题界面：**
- 上方：题干（"以下哪个表达最适合描述……"）
- 中间：4 个选项（单选）
- 选择后即时显示对错，不调 AI

**写作题界面：**
- 上方：写作场景描述 + 要求（如"请使用至少 2 个情绪类语料"）
- 中间：大型输入框
- 提交后 AI 纠错

### 5.4 AI 精细纠错引擎

用户提交翻译/写作答案后，AI 一次调用完成以下四维评估：

```
AI 输入：
  - 中文原题
  - 标准英文答案
  - 语料备注（如有）
  - 用户答案

AI 输出（JSON）：
{
  "verdict": "correct" | "partial" | "incorrect",
  "grammar_errors": [
    { "location": "...", "error": "...", "suggestion": "..." }
  ],
  "word_choice_issues": [
    { "original": "...", "issue": "...", "alternative": "..." }
  ],
  "improved_version": "AI润色后的参考译文"
}
```

**前端反馈展示：**
- 整体判断：顶部色条（绿色 ✅ / 黄色 ⚠️ / 红色 ❌）
- 语法错误：列表形式，原文错误词高亮
- 用词不当：对比展示用户用词 vs 推荐用词
- 改进版本：折叠面板，点击展开完整润色译文

**选择题判分：** 不需 AI，前端直接比对标准答案。

### 5.5 AI 适配器模块

```java
public interface AIService {
    List<Question> generateQuestions(QuestionRequest request);
    Feedback evaluateAnswer(EvaluateRequest request);
}
```

**实现类：**
- `OpenAICompatService`：调用任何兼容 OpenAI 接口的模型（DeepSeek、通义千问、智谱、Moonshot 等），通过配置 baseUrl + apiKey + modelName 切换
- `OllamaService`：调用本地 Ollama 模型

前端在练习页面提供 AI 后端下拉选择器，用户可随时切换。配置项（包括 API Key）存储在 `ai_provider` 表中。

### 5.6 学习追踪与仪表盘

**仪表盘页面包含：**

| 区域 | 内容 | 图表类型 |
|------|------|---------|
| 概览卡片 | 总练习次数、总题数、正确率、学习天数 | 数值卡片 |
| 正确率趋势 | 近 30 天每日正确率 | ECharts 折线图 |
| 分类正确率 | 各 subcategory 的正确率对比 | ECharts 柱状图 |
| 平均耗时 | 各分类平均答题耗时 | ECharts 柱状图 |
| 薄弱语料 Top 10 | 错误次数最多的 10 条语料 | 表格列表 |

数据全部从 `practice_answer` 和 `practice_session` 表实时查询统计。

## 8. 错误处理

- **AI 调用超时/失败**：前端给出友好提示"AI 服务暂时不可用，请稍后重试或切换其他 AI 后端"，题目保留可稍后重试
- **Excel 格式错误**：导入时逐行验证，跳过格式异常行并汇总报告
- **数据库异常**：Spring Boot 全局异常处理，返回统一 JSON 错误格式
- **API Key 无效**：在 AI 后端配置页提供"测试连接"按钮，保存前验证

---

## 9. 非功能需求

- **单用户优先**：当前不实现登录/鉴权，后续如需多用户再加（预留用户 ID 字段在各主表）
- **响应时间**：翻译题即时生成（<100ms），AI 生成题 <15s，AI 纠错 <10s
- **数据安全**：API Key 加密存储（AES），题库本地缓存降低 API 调用成本
